OOP Counter with State-Driven UI

Overview

The Counter component is a JavaScript class designed to manage a numeric value and its related user interface controls. It captures the counter’s state and behavior within a single object, allowing the component to handle user interactions, state updates, and DOM rendering internally.
The component provides controls to increment, decrement, and reset the current value while ensuring the count never drops below zero. When the value reaches zero, the decrement and reset buttons automatically become inactive through a conditional CSS class, providing clear visual feedback to the user.
Each counter instance maintains its own state, allowing multiple independent counters to be instantiated on the same page without conflicts.
An additional StepCounter class extends the base Counter component to demonstrate inheritance and polymorphism. This extended component introduces a configurable step parameter that determines how much the counter increases or decreases during each action.
Together, these components demonstrate key JavaScript concepts including encapsulation, class inheritance, state-driven UI updates, and modular component design.

How to Use It

The Counter component is designed to be easily instantiated inside any HTML container. Each counter manages its own state and UI, allowing multiple counters to exist independently on the same page.
To use the component, create a container element in your HTML and then instantiate the Counter class in JavaScript, passing the container selector and optional parameters.

Example JavaScript Instantiation Code

Each instance operates independently and manages its own state and DOM updates.

import Counter from "./Counter.js";
import StepCounter from "./StepCounter.js";
const counter1 = new Counter("#counter1");
const counter2 = new Counter("#counter2", 5);
const counter3 = new StepCounter("#counter3", 10, 2);
